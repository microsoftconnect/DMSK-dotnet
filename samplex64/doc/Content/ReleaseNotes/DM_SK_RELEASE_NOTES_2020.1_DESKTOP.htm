Release Notes for Dragon Medical SpeechKit 2020.1 (.NET edition)

New in version 2020.1

  • The version numbering for Dragon Medical SpeechKit, Dragon Medical Embedded and Dragon Medical Server has changed to a year.quarter scheme. The current Dragon Medical SpeechKit version is 2020.1.

Solved issues

  • Users who are not using PowerMic Mobile are no longer automatically assigned a PowerMic Mobile license. [DNB-23746]
  • Philips SpeechMike III: Pressing the fast-forward button to navigate to the next field no longer stops recording. [DMOCE-280]
  • Updated the Philips Speech Drivers delivered with the SDK. Installing only the server-side drivers no longer causes the application to stop responding. [CAVE-4055]
  • Corrected the character encoding for some error messages. [DMOCE-263]
  • Updated the information on supported microphones and button controls in the Dragon Medical SpeechKit Administration Guide. [DNB-25446]

Note: For information on previous releases, see: Release history.

Important information

  • Be aware that your application must be recompiled following this upgrade.
  • In partner integrations based on Dragon Medical SpeechKit (.NET edition), new users without a Nuance PowerMic Mobile license who select PowerMic Mobile from the Current Device menu are notified that they must apply for a license or select another input device. [DNB-17840]
  • To display your application's end user online help in the personalization & help window, make sure it can be loaded in an iframe. [DNB-20538]
  • The size of the .NET runtime package has increased, due to changes in the underlying technology, to support future product features and compatibility. [DNB-18566]
  • The VUI form focus is no longer set to the speech-enabled field when Initialize(), Open() or Synchronize() is called on the VuiController. [DNB-12643]
  • Filtering auto-texts (defining filters to deactivate auto-texts selectively) is deprecated and will no longer work.
  • The set of redistributable files was changed in version 3.4 release 1. Make sure to verify that all files in the bin folder are included with your application distribution.
  • Dragon Medical SpeechKit 2020.1 (.NET edition) requires Dragon Medical Server 3.3, 3.4, 3.5 or 2020.1.
  • Dragon Medical Server 2020.1 is compatible with client versions 3.3, 3.4, 3.5 and 2020.1.
  • Microsoft .NET Framework 4.7.1 (or higher) and Microsoft Internet Explorer 11 must be installed on all workstations running .NET edition-based applications.

Infrastructure

Infrastructure no longer supported

  • Dragon Medical SpeechKit and Dragon Medical Embedded 2020.1 applications do not work with version 3.1 and 3.2 servers.
  • Version 3.1 and 3.2 client applications do not work with version 2020.1 servers.
  • Microsoft Windows 7 is no longer supported.

Infrastructure changes planned for the next release

  • Dragon Medical SpeechKit and Dragon Medical embedded 2020.2 and higher will not work with version 3.3 servers.
  • Version 3.3 client applications will not work with version 2020.2 servers.
  • In the next release, Windows Embedded Standard 7 will no longer be supported.

Remarks

  • For more information on virtualized environments, see the Docs & Guidelines folder of the Nuance virtual extensions package on the Nuance Healthcare Development Platform.

  • Nuance Citrix/RDS/VMware Audio Extension and Nuance PowerMic Citrix/RDS/VMware Extension:

    The extensions do not need to be installed on the server/virtual desktop; the required server binaries are already included in the application folder.

    Make sure you uninstall the corresponding server/virtual desktop component setup if it is no longer used by other products.

  • Nuance RDS Audio Extension and Nuance PowerMic RDS Client Extension: UDP transport must be disabled on Remote Desktop Protocol (RDP) 8.1 clients and on RDP 10 clients. RDP 8.1 is preinstalled on Windows 8.1 and installed by Microsoft Update on Windows 7 and 8. RDP 10 is preinstalled on Windows 10.

    To disable UDP transport, add the following registry keys and value to the Remote Desktop Client, then restart the PC:

    Keys:

    HKEY_LOCAL_MACHINE\Software\Microsoft\Terminal Server Client

    HKEY_LOCAL_MACHINE\Software\Wow6432Node\Microsoft\Terminal Server Client

    DWORD Value:

    DisableUDPTransport = 1

  • The 64-bit version of the Nuance Citrix Client Audio Extension for Linux is now available.
  • VMware Horizon View 7.3: RTAV with device splitting is recommended if Nuance audio channels cannot be used.
  • To enable microphone button support for Philips, Grundig and Olympus devices, contact your device vendor for drivers that support your virtualized environment; deploy them on the server/virtual desktop where your application is hosted and on the client PC.

  • Microsoft RDS: When the Grundig drivers are not installed on both RDS server and client endpoint, the application using a Grundig device might become unresponsive when reconnecting to a session.

  • Philips SpeechMike on Linux thin clients in a Citrix environment: Session reconnect might not work. Solution: Make sure SpeechMike firmware version 2.54 or higher is installed.

*End-of-life: third-party environments

Support for third-party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third-party product and supporting Nuance documentation.

When standard support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s standard end-of-life and Nuance's policy. This means that issues that are a combination of the vendor's component and the Nuance solution cannot be supported.

Release history

New in version 3.5 release 1 (2019-11-12)

  • The microphone icons have been updated. [DNB-23359]
  • New methods SetCornerRadius and SetUIColor are available for customizing the speech bar. SetBackgroundImage is deprecated. For more information, see the Dragon Medical SpeechKit SDK Help. [DNB-20624]
  • The Philips SpeechOne PSM6000 headset is now supported. [DNB-22625]
  • Audio-related error messages are now more informative. [DNB-4126]

Solved issues in version 3.5 release 1

  • ITextControl events are no longer duplicated unnecessarily. [DNB-19253]
  • When the application is compiled with .NET 4.7.2, the options menu no longer cuts short the microphone list. This was solved by upgrading the Dragon Medical Server to a version that uses .NET 4.7.2. [DNB-21836]
  • The application no longer shuts down when the user unplugs a Nuance PowerMic III device and uses a connected PowerMic II device. [DNB-20663]
  • The "system is still loading" message displayed to users when recording cannot start is now properly localized. [DNB-10828]
