Release Notes for Dragon Medical SpeechKit 3.1 Service Release 10 (.NET edition)

Solved issues in service release 10

  • The application no longer shuts down unexpectedly in Citrix environments. [DNB-9572]
  • If a custom control's GetText method returns null, the application no longer shuts down unexpectedly. [DNB-5401]
  • Text is no longer inserted in the wrong location; this only occurred in rare circumstances. [DNB-9212]

Note: For more information on previous service releases, see: Service release history.

Important information

  • When recording is started, the GUI focus is no longer moved to the control that will receive the recognition results. [DNB-8791]
  • The set of redistributable files has changed in version 3.1 service release 3. Make sure to verify that all files in the bin folder are included with your application distribution.
  • Please be aware that your application must be recompiled following this upgrade.
  • Dragon Medical SpeechKit 3.1 (.NET edition) is compatible with server versions 2.1, 3.0 and 3.1
  • Dragon Medical Server 3.1. is compatible with client versions 3.0 and 3.1.
  • Microsoft .NET Framework 4.5 and Microsoft Internet Explorer 11 must be installed on all workstations running .NET edition-based applications. [DNB-4701]

What's new in version 3.1

Speech recognition

  • Speech recognition now uses neural network (deep learning) technology. (Only some languages are supported.)

Dragon Medical Analytics

  • The Server Management page is now displayed as a carousel in kiosk mode.
  • It is now possible to start Dragon Medical Analytics directly from the Nuance Management Center; this only applies to Nuance-hosted systems.
  • Dragon Medical Server configuration changes and PHI data access are now audited and accessible via the Nuance Management Server Auditor.
  • It is now possible to search for session logs based on the document ID provided by the partner.

User Interface

  • The personalization & help window has been redesigned to improve usability and to prepare it for additional features.
  • Auto-texts with lists are now supported as part of Dragon Medical PowerPacks. The auto-text contains variable text linked to a list of content options.
  • The speech recognition on/off icons now differ in both color and shape to support people with color vision deficiency.

On-premises systems

  • Dragon Medical Server can now connect to the Microsoft SQL Server using SQL Server Authentication.
  • Dragon Medical Server now supports Microsoft Windows Server 2016. Be aware that Nuance Management Server does not support Windows Server 2016.
  • Italian is now available as a user interface language.
  • Italian is now available as a speech recognition language for non-SpeechMagic-based systems.

Citrix environments

  • Nuance Citrix Server & Virtual Desktop Audio Extension is no longer available; the server binaries required are included in the application folder. Nuance Citrix Client Audio Extension does not need to be installed on the server/virtual desktop.
  • The Citrix extension deployment is now supported in a mixed environment (a range of client end points, some of which do not support the Nuance virtual extensions; for example, Linux thin clients and Windows clients).

Known issues

  • Recording is sometimes stopped or delayed when using IGEL clients in Citrix environments. [CAVE-1948]

Infrastructure

Infrastructure no longer supported

  • Version 3.1 client applications do not work with version 1.6 servers.
  • In Dragon Medical Server 3.1 all version 1.5 client-based applications stop working.

Infrastructure changes planned for the next release

  • In Dragon Medical Server 3.2 all version 1.6 and 2.0 client-based applications will stop working.
  • All applications based on Dragon Medical SpeechKit 3.2 will stop working with 2.0 servers.

Remarks

  • For more information, see the Nuance virtual extensions package on the Nuance Healthcare Development Platform.
  • Citrix environments: For Nuance PowerMic button support, install the Nuance PowerMic Citrix Extension on the Citrix client and server/virtual desktop where your application is hosted.

  • Microsoft RDS environments: For Nuance PowerMic button support, install the Nuance PowerMic RDS Client Extension on the Remote Desktop Client. Be aware of the following:

    The extension does not need to be installed on the server/virtual desktop; the required server binaries are already included in the application folder.

    The 64-bit and 32-bit versions of the Nuance PowerMic RDS Client Extension are installed together.

  • VMware Horizon Client 4.3.0 or higher: Install the new 64-bit Nuance PowerMic VMware Client Extension on the VMware client.
  • For microphone button support, third-party drivers for Philips, Grundig and Olympus devices need to be provided by the device vendor and deployed both on the Citrix/RDS client and Citrix/RDS server.
  • Nuance RDS Audio Extension on Windows Server 2012: Microsoft hotfix KB2802311 must be installed. For more information, see the Microsoft Knowledge Base, article 2802311. [PSP00069978]

  • Nuance RDS Audio Extension: UDP transport must be disabled on Remote Desktop Protocol (RDP) 8.1 and 10 clients. RDP 8.1 is preinstalled on Windows 8.1 and installed by Microsoft Update on Windows 7 and 8. RDP 10 is preinstalled on Windows 10. To disable UDP transport, add the following registry value to the Remote Desktop Client:

    Key: HKEY_LOCAL_MACHINE\Software\Microsoft\Terminal Server Client

    Value: DisableUDPTransport = 1

  • Philips SpeechMike on Linux thin clients in a Citrix environment: Session reconnect might not work. Solution: Make sure SpeechMike firmware version 2.54 or higher is installed.
  • For Microsoft Windows 10 clients Citrix Receiver 4.3 or higher is required.
  • For more information on virtualized environments, see the Docs & Guidelines folder of the Nuance virtual extensions package.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s and Nuance's end-of-life policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.

Dragon Medical SpeechKit product life cycle

Nuance provides support throughout the life cycle of one version and point release. Please note that Nuance will not perform problem analysis after the scheduled start of restricted maintenance. Dragon Medical Server versions released over three months after the dates stated here will not work with these client editions. For up-to-date information on these schedules, go to Nuance iSupport.

Service release history

Solved issues in service release 3

  • A Windows Presentation Foundation-based application no longer shuts down unexpectedly if a shared style in App.xaml is defined. [DNB-8379]
  • No memory leaks now occur when initializing and closing the VuiController. [DNB-7704]
  • If a session is not explicitly closed, Dragon Medical SpeechKit no longer causes unnecessary network traffic when no recording is running. [DNB-7836]
  • The keyboard focus no longer moves after recording is started. [DNB-8575]
  • On the speech bar, the Nuance flame icon was replaced by the hamburger icon. Clicking it now directly opens the personalization & help page menu. [DNB-6800]
  • Speech bar customization now works properly in Windows Presentation Foundation. [DNB-8612]
