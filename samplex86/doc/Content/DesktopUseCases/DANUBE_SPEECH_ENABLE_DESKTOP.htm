Speech-enabling your .NET desktop app

Important: If you're deploying Dragon Medical Server at your site (and not hosted by Nuance), make sure to configure Dragon Medical SpeechKit for on-premises deployment before speech-enabling your app.

Make sure to follow this procedure for all GUI forms that you want speech-enabled. By default all supported text controls on a GUI form that aren't disabled, hidden, read-only or password fields are speech-enabled.

Prerequisites

You've created one or more GUI forms and the session has been initialized (see: Speech recognition session initialization).

Procedure

To speech-enable a GUI form, do the following:

  1. Create a new VuiController object.
  2. Call Open() to initialize the VuiController, passing the container control you want speech-enabled as a parameter.
  3. Call Close() on the VuiController when the GUI form is closed.
  4. When all VuiControllers are closed and you don't intend to open additional windows with associated VuiControllers, call Session.SharedSession.Close() to close the session and fully disconnect from the Dragon Medical Recognition Service.
