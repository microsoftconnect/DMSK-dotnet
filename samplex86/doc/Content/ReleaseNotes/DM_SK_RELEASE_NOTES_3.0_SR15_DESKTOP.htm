Release Notes for Dragon Medical SpeechKit 3.0 Service Release 15 (.NET edition)

Dragon Medical SpeechKit is a component of the Nuance Healthcare Development Platform. Rebranding the components is currently an ongoing activity. Dragon Medical SpeechKit was previously called 360 | SpeechAnywhere Services; please be aware that references to 360 | SpeechAnywhere Services 3.0 apply to this release.

Solved issues in service release 15

  • Very short recordings are now recognized correctly. [DNB-7615]
  • Speech recognition no longer stops working when the form structure is changed and the VuiController activated/deactivated in a specific sequence. [DNB-7716]
  • When users record how they pronounce a word on the Manage Vocabulary page, the recording time limit of five seconds is now precisely enforced (requires Dragon Medical Server 3.1). [DNB-6007]

Note: For information on previous service releases, see: Service release history.

Important information

  • Please be aware that your application must be recompiled following this upgrade.
  • Dragon Medical SpeechKit 3.0 (.NET edition) is compatible with server versions 2.0, 2.1 and 3.0.
  • Dragon Medical Server 3.0 is compatible with client versions 2.0, 2.1 and 3.0.
  • Microsoft .NET Framework 4.5 must be installed on all workstations running .NET edition-based applications.
  • The .NET edition requires Microsoft Internet Explorer 11 to display the personalization and help window.
  • Information on the voice commands changed with each version of Dragon Medical SpeechKit is now available in the end user help.
  • Synchronize() can no longer be used instead of Initialize().
  • On the speech bar the colors for recording on/off have changed from red (on) - gray (off) to green (on) - red (off).

New in version 3.0

Manage Vocabulary (server version 3.0 required)

Dragon-based systems
  • Users can now search for words on the Manage Vocabulary page.
  • Users can now filter words on the Manage Vocabulary page based on the level (user/group/site) for which they were added.
  • New Modify Word screen: Users can now edit the spelling and pronunciation of words in their personal lexicon.
  • Users can now record how a word is pronounced.
  • New voice command to record how a word is pronounced: train word.

Personal auto-texts (server version 3.0 required)

All systems
  • Users can now create auto-texts via the personalization and help window, Manage Auto-texts page. This new functionality is available to desktop users and mobile device users with a Dragon Medical One Desktop Application/Dragon Medical Direct license.
  • New voice command: make that an auto-text.

Speech recognition (server version 3.0 required)

Dragon-based systems
  • New English, Dutch, Flemish and German voice commands to form compound words. English examples: compound, compound that, compound selection.
  • New German voice commands to separate compound words: trenne das, Auswahl trennen, trenne Auswahl.
  • Improved correction alternatives for Dutch, French and German in the correction menu.
  • Dragon Medical now intelligently identifies sentence separators; this means voice commands for selecting sentences and navigating to sentences work more reliably.
All systems
  • New voice commands to navigate text: go to beginning of sentence, go to end of sentence, go to beginning of paragraph, go to end of paragraph.

.NET edition

  • Dragon Medical SpeechKit performance has been improved in high latency situations.

On-premises systems (server version 3.0 required)

  • Dragon Medical systems now remove speech recognition data from the system file share when the data is no longer needed for improving the recognition quality.

Known issues

  • The automatic gain control (AGC) of the recording device or the device drivers/sound card drivers interferes with the Dragon Medical SpeechKit automatic audio adjustment feature. Solution: Disable the interfering AGCs.
  • The SpokenText parameter in the CommandRecognized event is empty in command mode. [DNB-328]
  • Shadow copying is not supported for applications integrating the .Net SDK. Dragon Medical SpeechKit will not work if shadow copying is enabled. [DNB-5181]

Infrastructure

Infrastructure no longer supported

  • Version 3.0 client applications do not work with version 1.5 servers.
  • In version 3.0, the .NET edition requires Microsoft Internet Explorer 11 to display the personalization and help window.
  • In Dragon Medical Server 3.0 all version 1.4 client-based applications stop working.

Infrastructure changes planned for the next release

  • In Dragon Medical Server 3.1 all version 1.5 and 1.6 client-based applications will stop working.
  • All applications based on Dragon Medical SpeechKit 3.1 will stop working with 360 | SpeechAnywhere Services 1.6 servers.

Remarks

  • For full support of virtualized environments, a custom audio channel must be deployed on the virtual server and client end point. For more information, see the Nuance virtual extensions package on the Nuance Healthcare Development Platform.

  • The Nuance virtual extensions were previously released as the SpeechMagic Citrix Extension, the SpeechMagic WTS/RDS Extension and the SpeechMagic VMware View Extension. The installation procedure has changed as compared to previous versions:

    Each extension now has a separate installer for the server/virtual desktop component and the client component.

    The 64-bit and 32-bit versions of each server/virtual desktop component are now installed together.

  • Backward compatibility: The Nuance virtual extensions client and server components are designed to be backward compatible from the virtual server to the client end point. This means that you can run newer versions of the Nuance virtual extensions on the virtual server and they will work with older versions of the corresponding client-side components with the following caveats and recommendations:

    New server-side components are backward compatible with the corresponding client components released within the last year.

    This policy was introduced with Nuance virtual extensions version 25.0.22.1; it does not apply to earlier releases.

    New releases contain bug fixes and optimizations and it is recommended to install the latest version.

  • Citrix environments: For Microsoft Windows 10 clients Citrix Receiver 4.3 or higher is required.
  • Citrix environments: For Nuance PowerMic button support, install the Nuance PowerMic Citrix Extension on the Citrix client and server/virtual desktop where your application is hosted.

  • Microsoft RDS environments: For Nuance PowerMic button support, install the Nuance PowerMic RDS Client Extension on the Remote Desktop Client. Be aware of the following:

    The extension does not need to be installed on the server/virtual desktop; the required server binaries are already included in the application folder.

    The 64-bit and 32-bit versions of the Nuance PowerMic RDS Client Extension are installed together.

  • VMware Horizon View/RDSH environments: For Nuance PowerMic button support, install the Nuance PowerMic VMware Client Extension on the client PC. The extension does not need to be installed on the server/virtual desktop; the required server binaries are already included in the application folder.
  • For microphone button support, third-party drivers for Philips, Grundig and Olympus devices need to be provided by the device vendor and deployed both on the Citrix/RDS client and Citrix/RDS server.
  • Nuance RDS Audio Extension on Windows Server 2012: Microsoft hotfix KB2802311 must be installed. For more information, see the Microsoft Knowledge Base, article 2802311. [PSP00069978]

  • Nuance RDS Audio Extension: UDP transport must be disabled on Remote Desktop Protocol (RDP) 8.1 and 10 clients. RDP 8.1 is preinstalled on Windows 8.1 and installed by Microsoft Update on Windows 7 and 8. RDP 10 is preinstalled on Windows 10. To disable UDP transport, add the following registry value to the Remote Desktop Client:

    Key: HKEY_LOCAL_MACHINE\Software\Microsoft\Terminal Server Client

    Value: DisableUDPTransport = 1

  • Citrix XenDesktop: Third-party drivers for Olympus devices are not available at the time of the Dragon Medical SpeechKit 3.0 release. Contact your device vendor for updated drivers with XenDesktop support.
  • Philips SpeechMike on Linux thin clients in a Citrix environment: Session reconnect might not work. Solution: Make sure SpeechMike firmware version 2.54 or higher is installed.
  • VMware environments: For more information on system configuration, see the VMware Knowledge Base (Article ID 2068447).
  • VMware Virtual Audio on VMware virtual desktops running Microsoft Windows 8.1: Recording can sometimes stop with an internal audio error. This is due to an issue with VMware virtual machines and the xHCI USB host controller for Windows 8.1. Solution: Remove the xHCI controller from the virtual machine and add the EHCI+UHCI controller. For more information, see the VMware Knowledge Base, article 2057809. [PSP00073153]
  • For more information on virtualized environments, see the Docs & Guidelines folder of the Nuance virtual extensions package.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s and Nuance's end-of-life policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.

Dragon Medical SpeechKit product life cycle

Nuance provides support throughout the life cycle of one version and point release. Please note that Nuance will not perform problem analysis after the end-of-life date stated here. If a Nuance-hosted system is used, Nuance cannot guarantee that an older version will continue working after the date stated here.

Service release history

Solved issues in service release 2

  • The application no longer stops responding when the VUIController is closed at the same time as the start of a new utterance. [DNB-6290]
  • ControlDeviceEventReceived callbacks are now invoked on the session callback thread. [DNB-6464]
  • The user no longer has to click a field twice to set the keyboard focus when a different application had the focus. [DNB-6620]
  • In Citrix environments, a black rectangle no longer stays on the screen after an error message has been displayed. [DNB-6276]
  • Nuance PowerMic III: Applications now display the correct device name when the PowerMic III is connected (PowerMic III is still identified as PowerMic II by the operating system). [CAVE-847]

New in version 3.0 service release 2

  • Nuance PowerMic button controls are now supported in Microsoft RDS environments.

Solved issues in service release 7

  • Error recovery and support for load balancing have been improved. [DNB-6889]
  • After a session is closed and recording is no longer possible, the Nuance flame is no longer displayed in speech-enabled fields. [DNB-6673]
  • The application no longer stops unexpectedly during initialization with the latest .NET Framework version. [DNB-6976]

New in version 3.0 service release 7

  • 360 | SpeechAnywhere Services client editions are now called Dragon Medical SpeechKit. [DNB-5670]

  • The speech recognition on/off icons now differ in both color and shape to support people with color vision deficiency. [DNB-6458]
  • The Record button on a control device is now synchronized with the Record button on the user interface. [DNB-6758]

Solved issues in service release 12

  • Changing the values of a set of application-defined placeholders (leaving the IDs and labels unchanged) via AssignCommandPlaceholders() no longer causes the application to stop unexpectedly. [DNB-7414]
  • Adding an application command to an existing command set via CreateCommand() now works correctly; the new command is no longer ignored. [DNB-7451]
  • Improved speech recognition if a user is dictating very quietly. [CAVE-857]

New in version 3.0 service release 12

  • The error notification has a new look. [DNB-6623]
  • The sample projects are now targeted for Visual Studio 2015. [DNB-5603]
  • Support for VMware Horizon View published applications (RDSH). [CAVE-252]
