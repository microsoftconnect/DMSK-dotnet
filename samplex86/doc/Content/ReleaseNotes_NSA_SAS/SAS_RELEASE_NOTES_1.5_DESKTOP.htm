Release Notes for 360 | SpeechAnywhere Services 1.5 (.NET edition)

Important information

  • Please be aware that your application must be recompiled following this upgrade.
  • 360 | SpeechAnywhere Services 1.5 Server is compatible with client versions 1.3, 1.4 and 1.5.
  • 360 | SpeechAnywhere Services 1.5 will work on Microsoft Windows XP, however please be aware that newer operating systems are recommended due to the Microsoft support policy.

Known issues

  • The automatic gain control (AGC) of the recording device or the device drivers/sound card drivers interferes with the SpeechAnywhere Services automatic audio adjustment feature. Solution: disable the interfering AGCs.
  • The SpokenText parameter in the CommandRecognized event is empty in command mode. [PSP00062085]

Infrastructure

Infrastructure no longer supported

  • Version 1.1 client-based applications will stop working.

Infrastructure changes planned for the next release

  • In 360 | SpeechAnywhere Services 1.6 all version 1.2 client-based applications will stop working.
  • In 360 | SpeechAnywhere Services 1.6 all Microsoft .NET 3.5-based applications will stop working. The minimum requirement will be Microsoft .NET 4.0.
  • In 360 | SpeechAnywhere Services 1.6, Microsoft Windows XP will no longer be supported.
  • In 360 | SpeechAnywhere Services 1.6, Microsoft Windows XP will no longer be supported as the Remote Desktop Client.
  • In 360 | SpeechAnywhere Services 1.6, Citrix XenApp 6.0 will no longer be supported.

Infrastructure changes planned for the release after next

  • In 360 | SpeechAnywhere Services 1.7 all Microsoft .NET 4.0-based applications will stop working. The minimum requirement will be Microsoft .NET 4.5.

Remarks

  • Virtual environments: For full support of virtual platforms, a custom audio channel needs to be deployed both on the platform client and platform server. For more information, see the custom audio channel deployment package in the SpeechMagic package or on the Nuance Healthcare Development Platform.
  • Citrix and Microsoft RDS environments: For microphone button support, third-party drivers for Philips, Grundig and Olympus devices need to be provided by the device vendor and deployed both on the platform client and platform server. For Nuance PowerMic devices, these drivers also need to be deployed and are included with the SpeechMagic package or with the driver package on the Nuance Healthcare Development Platform.
  • For Citrix XenDesktop, third-party drivers for Philips, Grundig and Olympus devices are not available at the time of the 360 | SpeechAnywhere Services 1.5 release. Please contact your device vendor for updated drivers with XenDesktop support.
  • For VMware environments, please find more information about system configuration in the VMware Knowledge Base.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations the vendor’s end-of-life policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.
