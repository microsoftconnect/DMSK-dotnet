Release Notes for 360 | SpeechAnywhere Services 1.6 Service Release 11 (.NET edition)

Solved issues in service release 11

  • Applications are no longer prevented from reconnecting to the SAS Server after a session timeout in suboptimal network environments. [PSP00075746, PSP00078705]
  • Dragon-based systems: The handling of text deletion while recognition results are pending has been improved. [PSP00076130]
  • Adding a natively-supported text control to a form and immediately enabling it after recognition has been performed in a custom control no longer causes the application to stop responding. [PSP00077699]
  • When multiple instances of an application are open, it is now clearer which speech bar corresponds to the instance with the focus. [PSP00076672]
  • The speech bar is no longer displayed off-screen in some circumstances when the application starts. [PSP00078259]
  • The speech bar no longer removes the focus from an application when clicked. [PSP00077841]
  • The design of the speech bar has been improved to work better in Citrix environments. [PSP00074956]
  • Information has been added to the .NET edition SDK Help on how to receive command events. [PSP00070981]

Note: For information on previous service releases, see below.

New in version 1.6 service release 11

  • The design of the speech bar and options menu has been improved. [PSP00077120, PSP00079184]

  • The speech bar is now customizable. Your integration can set the background color, background image, border image, button images and start position. To access the speech bar functionality, the .NET edition SDK has been extended with a new Windows Forms API and a new WPF API. The types and conventions of the new APIs match those of their respective UI frameworks but they are otherwise identical. Either can be used in any application. Windows Forms API example:

    Setting the initial position of the speech bar so that the upper left corner aligns to the (100, 100) position:

    SpeechBar.SharedSpeechBar.SetInitialPosition(100, 100, true, true);

    Overriding the default button backgrounds (assuming the images are resources of the application):

    SpeechBar.SharedSpeechBar.SetBackgroundImage(Nuance.SpeechAnywhere.BackgroundImageType.RecordButtonOffNormal, Properties.Resources.recbutton_default);

    SpeechBar.SharedSpeechBar.SetBackgroundImage(Nuance.SpeechAnywhere.BackgroundImageType.RecordButtonOffHover, Properties.Resources.recbutton_default_hover);

    For more information, see the API reference documentation and the sample application. [PSP00077120, PSP00075828, PSP00062206, PSP00079341]

  • Commands and actions can now be assigned to all available function buttons on Philips SpeechMike III and SpeechMike Premium devices. The ControlDeviceEvent enumeration has been extended to include values Button4Pressed, Button5Pressed, up to Button8Pressed. [PSP00078139]

Note: For information on previous service releases, see below.

Important information in version 1.6

  • Please be aware that your application must be recompiled following this upgrade.
  • 360 | SpeechAnywhere Services 1.6 Server is compatible with client versions 1.4, 1.5 and 1.6.
  • 360 | SpeechAnywhere Services 1.6 (.NET edition) is compatible with SAS Server versions 1.4, 1.5 and 1.6.
  • The ErrorOccurred event is deprecated and will be removed in 360 | SpeechAnywhere Services 1.7.

New in version 1.6

  • Voice editing functionality has been extended with select & say voice commands (Dragon-based systems only).
  • Text formatting has been improved. When text is inserted, the spacing is correctly updated and surrounding text is correctly capitalized (Dragon-based systems only).
  • SAS clients can now use Dragon customization packs (.dat files) containing Text-and-Graphics/auto-texts without lists. You can import and share .dat files via the Nuance Management Console.
  • The .NET edition has a new speech bar with an options button, a volume meter and a start/stop recording button. This replaces the speech bubble from earlier versions. It can be freely positioned by the end user. It remains on the Windows desktop when the speech-enabled application is minimized.
  • The .NET edition has a new custom text control interface; it is now possible to speech-enable any text control in your application.
  • The .NET edition now supports TX TextControl .NET and TX TextControl WPF.

Known issues

  • The automatic gain control (AGC) of the recording device or the device drivers/sound card drivers interferes with the SpeechAnywhere Services automatic audio adjustment feature. Solution: Disable the interfering AGCs.
  • The SpokenText parameter in the CommandRecognized event is empty in command mode. [PSP00062085]

Infrastructure

Infrastructure no longer supported

  • Version 1.2 client applications will stop working with version 1.6 servers.
  • All Microsoft .NET 3.5-based applications will stop working. The minimum requirement is Microsoft .NET 4.0.
  • Microsoft Windows XP is no longer supported.
  • Microsoft Windows XP is no longer supported for the Remote Desktop client.
  • Citrix XenApp 6.0 is no longer supported.

Infrastructure changes planned for the next release

  • In 360 | SpeechAnywhere Services 1.7 all version 1.3 client-based applications will stop working.
  • In 360 | SpeechAnywhere Services 1.7 all Microsoft .NET 4.0-based applications will stop working. The minimum requirement will be Microsoft .NET 4.5.

Remarks

  • Virtual environments: For full support of virtual platforms, a custom audio channel needs to be deployed both on the platform client and platform server. For more information, see the custom audio channel deployment package in the SpeechMagic package or on the Nuance Healthcare Development Platform.
  • Citrix and Microsoft RDS environments: For microphone button support, third-party drivers for Philips, Grundig and Olympus devices need to be provided by the device vendor and deployed both on the Citrix/RDS client and Citrix/RDS server. For Nuance PowerMic devices, drivers also need to be deployed on the Citrix/RDS client and server and are included with the SpeechMagic package or with the driver package on the Nuance Healthcare Development Platform.
  • For Citrix XenDesktop, third-party drivers for Philips, Grundig and Olympus devices are not available at the time of the 360 | SpeechAnywhere Services 1.6 release. Contact your device vendor for updated drivers with XenDesktop support.
  • VMware environments: For more information on system configuration, see the VMware Knowledge Base (Article ID 2068447).

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s and Nuance's end-of-life policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.
