Release Notes for Dragon Medical SpeechKit 3.4 Release 2 (.NET edition)

New in version 3.4 release 2

  • Philips SpeechMike Premium Touch SMP3700/3800 is now supported. [DNB-12008]

Solved issues

  • Applications based on Dragon Medical SpeechKit 3.4 now work with Dragon Medical Server version 3.1. [DNB-16343, DNB-17745]
  • WPF controls: Cross-thread issues no longer occur when the speech bar and an another integration resource access a style with the same name. [DNB-15539]

Note: For information on previous releases, see: Release history.

Important information

  • The size of the .NET runtime package has increased, due to changes in the underlying technology, to support future product features and compatibility. [DNB-18566]
  • The VUI form focus is no longer set to the speech-enabled field when Initialize(), Open() or Synchronize() is called on the VuiController. [DNB-12643]
  • VMware environments: The VMware Blast Extreme protocol is now fully supported.
  • Filtering auto-texts (defining filters to deactivate auto-texts selectively) is deprecated and will no longer work.
  • The set of redistributable files was changed in version 3.4 release 1. Make sure to verify that all files in the bin folder are included with your application distribution.
  • Please be aware that your application must be recompiled following this upgrade.
  • Dragon Medical SpeechKit 3.4 Release 2 (.NET edition) requires Dragon Medical Server 3.1, 3.2, 3.3 or 3.4.
  • Dragon Medical Server 3.4 is compatible with client versions 3.1, 3.2, 3.3 and 3.4.
  • Microsoft .NET Framework 4.5 and Microsoft Internet Explorer 11 must be installed on all workstations running .NET edition-based applications.

Infrastructure

Infrastructure no longer supported

  • Dragon Medical SpeechKit and Dragon Medical embedded 3.4 applications do not work with version 3.0 servers.
  • Version 3.0 client applications do not work with version 3.4 servers.

Infrastructure changes planned for the next release

  • Dragon Medical SpeechKit and Dragon Medical embedded 3.5 applications will not work with version 3.1 and 3.2 servers.
  • Version 3.1 and 3.2 client applications will not work with version 3.5 servers.

Remarks

  • For more information on virtualized environments, see the Docs & Guidelines folder of the Nuance virtual extensions package on the Nuance Healthcare Development Platform.
  • VMware Horizon View 7.3: RTAV is recommended if Nuance audio channels cannot be used.
  • Citrix environments: For Nuance PowerMic button support, install the Nuance PowerMic Citrix Extension on the Citrix client and server/virtual desktop where your application is hosted.

  • Microsoft RDS environments: For Nuance PowerMic button support, install the Nuance PowerMic RDS Client Extension on the Remote Desktop Client. Be aware of the following:

    The extension does not need to be installed on the server/virtual desktop; the required server binaries are already included in the application folder.

    The 64-bit and 32-bit versions of the Nuance PowerMic RDS Client Extension are installed together.

  • For microphone button support, third-party drivers for Philips, Grundig and Olympus devices need to be provided by the device vendor and deployed both on the Citrix/RDS client and Citrix/RDS server.
  • Nuance RDS Audio Extension on Windows Server 2012: Microsoft hotfix KB2802311 must be installed. For more information, see the Microsoft Knowledge Base, article 2802311. [PSP00069978]

  • Nuance RDS Audio Extension: UDP transport must be disabled on Remote Desktop Protocol (RDP) 8.1 and 10 clients. RDP 8.1 is preinstalled on Windows 8.1 and installed by Microsoft Update on Windows 7 and 8. RDP 10 is preinstalled on Windows 10. To disable UDP transport, add the following registry value to the Remote Desktop Client:

    Key: HKEY_LOCAL_MACHINE\Software\Microsoft\Terminal Server Client

    Value: DisableUDPTransport = 1

  • Philips SpeechMike on Linux thin clients in a Citrix environment: Session reconnect might not work. Solution: Make sure SpeechMike firmware version 2.54 or higher is installed.
  • For Microsoft Windows 10 clients Citrix Receiver 4.3 or higher is required.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When standard support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s standard end-of-life and Nuance's policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.

Release history

New in version 3.4 release 1 (2018-05-25)

  • When a user's workstation is locked, recording is stopped and pressing any microphone button will be ignored. [DNB-12596]

Solved issues in version 3.4 release 1

  • After a desktop recompose on Citrix Wyse clients, an internal audio error message is no longer displayed when the application is opened for the first time. [DNB-10176]
  • The application no longer shuts down unexpectedly in very rare cases when recording is stopped. [DNB-12256]

Solved issues in version 3.3 release 5 (2018-02-23)

  • Tab controls no longer sometimes switch to a previously active tab. [DNB-12643]
  • Line breaks and keyboard shortcuts now work properly in the personalization and help window. [DNB-13143]
  • Incorrect warning messages are no longer displayed. [DNB-12150, DNB-12454]
  • The application no longer becomes unresponsive in rare cases. [DNB-12943]

New in version 3.3 release 3 (2017-12-15)

  • The size of GUI elements no longer depends on screen resolution. [DNB-10676]

Solved issues in version 3.3 release 3

  • The application no longer shuts down unexpectedly when a recording device is unplugged. [DNB-10637]
  • The recording device menu on the speech bar is now updated correctly when a recording device is connected or disconnected. [DNB-11639]
  • Error messages are no longer displayed before the speech bar opens. [DNB-11009]
  • The default microphone sensitivity has been updated to improve system behavior. [DNB-12186]

New in version 3.3 release 1 (2017-11-10)

  • The recording device menu on the speech bar is now updated automatically when a recording device is connected or disconnected.
  • Users can now adjust the sensitivity of their microphone to reduce the effect of noisy environments (requires Dragon Medical Server 3.3 Release 1).
  • Users can now activate the press-and-hold mode to improve speech recognition results in noisy environments (requires Dragon Medical Server 3.3 Release 1).
  • When users select the microphone they want to use for dictation via the menu on the speech bar, their selection will be remembered even if they use a different workstation.

Solved issues in version 3.3 release 1

  • In partner integrations based on Dragon Medical SpeechKit (.NET edition), PowerMic Mobile now works properly for new users. [DNB-11621]
  • Special characters in concept names and line/paragraph breaks are now parsed correctly. [DNB-11817]
  • All fields are now speech-enabled correctly if the application contains multiple tabs. [DNB-10893]
  • The last milliseconds of an utterance are no longer split off in rare cases. [DNB-11699]
