Release Notes for 360 | SpeechAnywhere Services 2.1 (.NET edition)

Important information

  • Please be aware that your application must be recompiled following this upgrade.
  • 360 | SpeechAnywhere Services 2.0 Server is compatible with client versions 1.5, 1.6, 2.0 and 2.1.
  • 360 | SpeechAnywhere Services 2.1 (.NET edition) is compatible with server versions 1.6, 1.7, 1.8 and 2.0.

  • The ErrorOccurred event is deprecated and never fired in 360 | SpeechAnywhere Services 2.1.

  • From 360 | SpeechAnywhere Services Server 2.2, Microsoft Internet Explorer 10 and 11 will be required on .Net SDK to display the What You Can Say page, end user help and other functionality.

New in version 2.1

  • Simplified deployment in Citrix, RDS and VMware environments.

  • The package size of the .Net and COM SDK redistributables has been significantly reduced.

  • Field navigation starts a new utterance.
  • RDS and VMware deployment and configuration guides are now available.
  • Deployment and configuration guides for the Nuance virtual extensions are now available in French and German.
  • The handling of temporary network problems has been improved.
  • A VB.Net sample is now available.
  • Continuous recording across VUI form boundaries is now possible.

  • The integrator can now actively improve speech recognition; notifying SpeechAnywhere about the document correction state as well as the finalized version via the support functionality for document workflows. For more information, see: 'Support functionality for document workflows' in the SpeechAnywhere SDK help.

  • Grundig Digta SonicMic 3 is now supported.
  • Linux thin clients: Grundig Digta SonicMic II Classic and SonicMic II Classic US are now supported.
  • PowerMic III custom buttons are now supported.
  • Microsoft Windows 10 is now supported.

Known issues

  • The automatic gain control (AGC) of the recording device or the device drivers/sound card drivers interferes with the SpeechAnywhere automatic audio adjustment feature. Solution: Disable the interfering AGCs.
  • The SpokenText parameter in the CommandRecognized event is empty in command mode. [PSP00062085]

  • If the field with the focus is removed from the speech-enabled form and then added again, recognition results will be inserted into the wrong field until the next manual focus change. [DNB-4609]

Infrastructure

Infrastructure no longer supported

  • Version 2.1 client applications do not work with version 1.4 servers.

  • All Microsoft .NET 4.0-based applications will stop working. The minimum requirement will be Microsoft .NET 4.5.

Infrastructure changes planned for the next release

  • In 360 | SpeechAnywhere Services 2.2 all version 1.5 and 1.4 client-based applications will stop working.
  • In 360 | SpeechAnywhere Services 2.2 all version 2.2 client-based applications will stop working with version 1.5 servers.
  • In 360 | SpeechAnywhere Services Server 2.2, Microsoft Internet Explorer 10 and 11 will be required on .Net SDK to display the What You Can Say page, end user help and other functionality.

Remarks

  • Virtualized environments: For full support of virtual platforms, a custom audio channel needs to be deployed both on the platform client and platform server. For more information, see the custom audio channel deployment package on the Nuance Healthcare Development Platform.

  • Important: The virtual audio channels delivered with this package were previously released as the SpeechMagic Citrix Extension, the SpeechMagic WTS/RDS Extension and the SpeechMagic VMware View Extension. The installation procedure has changed as compared to previous versions: Each extension now has a separate installer for the server/virtual desktop component and the client component. The 64-bit and 32-bit versions of each server/virtual desktop component are now installed together.

  • For microphone button support, third-party drivers for Philips, Grundig and Olympus devices need to be provided by the device vendor and deployed both on the Citrix/RDS client and Citrix/RDS server. For Nuance PowerMic devices, drivers also need to be deployed on the Citrix client and server and are included with the driver package on the Nuance Healthcare Development Platform.
  • Nuance RDS Audio Extension on Windows Server 2012: Microsoft hotfix KB2802311 must be installed. For more information, see the Microsoft Knowledge Base, article 2802311. [PSP00069978]

  • Nuance RDS Audio Extension: UDP transport must be disabled on Remote Desktop Protocol (RDP) 8.1 clients. RDP 8.1 is preinstalled on Windows 8.1 and installed by Microsoft Update on Windows 7 and 8. To disable UDP transport, add the following registry value to the Remote Desktop client

    Key: HKEY_LOCAL_MACHINE\Software\Microsoft\Terminal Server Client

    Value: DisableUDPTransport = 1

  • For Citrix XenDesktop, third-party drivers for Philips, Grundig and Olympus devices are not available at the time of the 360 | SpeechAnywhere Services 1.6 release. Contact your device vendor for updated drivers with XenDesktop support.
  • VMware environments: For more information on system configuration, see the VMware Knowledge Base (Article ID 2068447).

  • VMware Virtual Audio on VMware virtual desktops running Microsoft Windows 8.1: Recording can sometimes stop with an internal audio error. This is due to an issue with VMware virtual machines and the xHCI USB host controller for Windows 8.1. Solution: Remove the xHCI controller from the virtual machine and add the EHCI+UHCI controller. For more information, see the VMware Knowledge Base, article 2057809. [PSP00073153]

  • For more information on virtualized environments, see the Docs & Guidelines folder of the Nuance virtual extensions package.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s and Nuance's end-of-life policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.
