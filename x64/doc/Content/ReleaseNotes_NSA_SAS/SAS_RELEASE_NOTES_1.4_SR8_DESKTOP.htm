Release Notes for 360 | SpeechAnywhere Services 1.4 Service Release 8 (.NET edition)

Solved issues

When the audio input is too loud the volume meter now correctly displays feedback in red. [PSP00069441]

In a Windows Presentation Foundation application, the VuiVontroller now always initializes correctly. [PSP00069848]

The .NET SDK no longer crashes when a VUI form is reinitialized. [PSP00070400]

The speech bar is now always visible when your application is set as 'always on top'. [PSP00069203]

Solved issues in service release 1

When recording is started the first 500 ms of audio is no longer lost. [PSP00069214]

Important information

  • 360 | SpeechAnywhere Services 1.4 Server is compatible with client versions 1.2 and 1.3.
  • SAS will work on Microsoft Windows XP, however please be aware that newer operating systems are recommended due to the Microsoft support policy.
  • The delete field voice command has been removed.
  • In SAS "license GUID" has changed to "organization token" and in NMC "customer token" has changed to "organization token".

Known issues

  • The automatic gain control (AGC) of the recording device or the device drivers/sound card drivers interferes with the SAS automatic audio adjustment feature. Solution: disable the interfering AGCs.
  • The SpokenText parameter in the CommandRecognized event is empty in command mode. [PSP00062085]

Infrastructure no longer supported

  • Version 1.0 client-based applications will stop working.

Infrastructure changes planned for the next release

  • In 360 | SpeechAnywhere Services 1.5 all version 1.1 client-based applications will stop working.

Infrastructure changes planned for future releases

  • In 360 | SpeechAnywhere Services 1.6 all Microsoft .Net 3.5-based applications will stop working. The minimum requirement will be Microsoft .Net 4.0.
