Release Notes - 360 | SpeechAnywhere Services 1.3 Service Release 8

This is a service release for 360 | SpeechAnywhere Services .NET edition.

This service release is cumulative and contains all changes to previous SAS .NET edition service releases.

Solved issues

Audio

In cases of high workstation CPU load or faulty audio devices, recording sometimes stopped but the user was not notified.

In a Citrix/WTS environment with an unreliable network connection, recording sometimes stopped but the user was not notified.

Network connection

Users sometimes experienced a delay of up to 5 minutes which resulted in a loss of connectivity and loss of parts of the dictation; this was due to a Microsoft issue in the networking stack (for more information, see: http://support.microsoft.com/kb/976816).

Previous service release information

Service release 2

The system sometimes stopped processing sound. The system now reliably processes sound and returns the corresponding recognized text.

Windows Terminal Server is now supported; the SpeechMagic WTS Extension must be installed.

The Voice Activity Detection (VAD) algorithm has been improved.

Active command sets can now be changed when the system is listening

If you have defined application commands with more than one display string all are now displayed on the What You Can Say page.

Known issues

Occasionally the connection to the USB recording device is lost and cannot be re-established. Workaround: plug the device into another USB port.

In hosted systems the first 2 utterances of a first time user are not reliably recognized.

In on-premises systems the SpokenText parameter of CommandRecognized events is always empty.
