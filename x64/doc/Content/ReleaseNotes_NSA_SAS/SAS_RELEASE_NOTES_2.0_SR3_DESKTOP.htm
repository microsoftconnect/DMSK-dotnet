Release Notes for 360 | SpeechAnywhere Services 2.0 Service Release 3 (.NET edition)

Solved issues in service release 3

  • WPF applications: Setting custom styles in the App.xaml file no longer prevents the speech bar from being displayed nor causes the application to stop responding. [DNB-3150]
  • Audio data from the start of the recording is no longer sometimes excluded from recognition. [DNB-1372]
  • Speech recognition no longer stops occasionally because of an inconsistency in the amount of audio data sent in one server request. [DNB-3143]
  • Calling Synchronize() on the VuiController immediately after recording has started for the first time no longer stops recording. [DNB-3151]
  • When incorrect custom control implementations report invalid Unicode characters to SpeechAnywhere Services, recording now stops and speech recognition becomes unavailable until the application is restarted. [DNB-3386]
  • Initializing a VuiController no longer activates the window whose controls are speech enabled by the VuiController. [DNB-2322]

Important information in version 2.0

  • Please be aware that your application must be recompiled following this upgrade.
  • 360 | SpeechAnywhere Services 2.0 Server is compatible with client versions 1.5, 1.6 and 2.0.
  • 360 | SpeechAnywhere Services 2.0 (.NET edition) is compatible with SAS Server versions 1.5, 1.6 and 2.0.
  • The ErrorOccurred event is deprecated and will be removed in 360 | SpeechAnywhere Services 2.1.

New in version 2.0

Dragon-based systems
  • Enhanced adaptation of users' profiles in Nuance-hosted systems, including Language Model adaptation.
  • The speech recognition error rate has been reduced by more than 10% compared to version 1.6.
  • Voice commands that format the next words spoken: all caps, all caps on, all caps off, no caps, no caps on, no caps off, no space, no space on, no space off, numeral and Roman numeral.
SpeechMagic-based systems:
  • Improved speech recognition results using statistical noise recognition.
All systems

  • The new SpeechAnywhere Analytics web application displays comprehensive information on your SpeechAnywhere system.

    In your web browser, go to:

    On-premises systems: http(s)://<your.SpeechAnywhere.Server>:<port>/SpeechAnalytics

    Nuance-hosted system: https://nsa.nuancehdp.com/SpeechAnalytics

    Log on with your Nuance Management Console (NMC) user name and password.

  • The What You Can Say page has been reworked, including expandable sections and new search functionality.
  • The Manage Vocabulary page has been enhanced; users can view their personal lexicon.
  • Embedded fields in auto-texts and navigation between fields.
  • New voice command: cap that.
  • Faster recognition results when the user first starts speaking.
  • Faster recognition results when there are many user-added words.
  • You can now use microphone buttons to navigate embedded fields in auto-texts (SpeechAnywhere Server 2.0 required).

Known issues in version 2.0

  • The automatic gain control (AGC) of the recording device or the device drivers/sound card drivers interferes with the SpeechAnywhere Services automatic audio adjustment feature. Solution: Disable the interfering AGCs.
  • The SpokenText parameter in the CommandRecognized event is empty in command mode. [PSP00062085]

Infrastructure

Infrastructure no longer supported

  • Version 1.3 client applications do not work with version 2.0 servers.
  • Version 2.0 client applications do not work with version 1.3 servers.

Infrastructure changes planned for the next release

  • In 360 | SpeechAnywhere Services 2.1 all version 1.4 client-based applications will stop working.
  • In 360 | SpeechAnywhere Services 2.1 all version 2.1 client-based applications will stop working with version 1.4 servers.
  • In 360 | SpeechAnywhere Services 2.1 all Microsoft .NET 4.0-based applications will stop working. The minimum requirement will be Microsoft .NET 4.5.

Remarks

  • Virtual environments: For full support of virtual platforms, a custom audio channel needs to be deployed both on the platform client and platform server. For more information, see the custom audio channel deployment package in the SpeechMagic package or on the Nuance Healthcare Development Platform.
  • Citrix and Microsoft RDS environments: For microphone button support, third-party drivers for Philips, Grundig and Olympus devices need to be provided by the device vendor and deployed both on the Citrix/RDS client and Citrix/RDS server. For Nuance PowerMic devices, drivers also need to be deployed on the Citrix/RDS client and server and are included with the SpeechMagic package or with the driver package on the Nuance Healthcare Development Platform.
  • For Citrix XenDesktop, third-party drivers for Olympus devices are not available at the time of the 360 | SpeechAnywhere Services 2.0 release. Contact your device vendor for updated drivers with XenDesktop support.
  • VMware environments: For more information on system configuration, see the VMware Knowledge Base (Article ID 2068447).
  • Nuance RDS Audio Extension on Windows Server 2012: Microsoft hotfix KB2802311 must be installed. For more information, see the Microsoft Knowledge Base, article 2802311. [PSP00069978]

  • Nuance RDS Audio Extension: UDP transport must be disabled on Remote Desktop Protocol (RDP) 8.1 clients. RDP 8.1 is preinstalled on Windows 8.1 and installed by Microsoft Update on Windows 7 and 8. To disable UDP transport, add the following registry value to the Remote Desktop client:

    Key: HKEY_LOCAL_MACHINE\Software\Microsoft\Terminal Server Client

    Value: DisableUDPTransport = 1

  • Philips SpeechMike with VMware Horizon View clients: If the Philips SpeechMike Extension for VMware View is installed with the Nuance VMware Audio Extension, SpeechMike buttons do not work. Solution: Do not install the Philips SpeechMike Extension; install the Nuance VMware Audio Extension and configure USB device splitting. For more information, see Deployment and Configuration for VMware Horizon View Environments (Docs & Guidelines folder) and the VMware Knowledge Base, article 2068447. [SM-364]
  • VMware Virtual Audio on VMware virtual desktops running Microsoft Windows 8.1: Recording can sometimes stop with an internal audio error. This is due to an issue with VMware virtual machines and the xHCI USB host controller for Windows 8.1. Solution: Remove the xHCI controller from the virtual machine and add the EHCI+UHCI controller. For more information, see the VMware Knowledge Base, article 2057809. [PSP00073153]
  • For more information, see the Docs & Guidelines folder of the Nuance virtual extensions package.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s and Nuance's end-of-life policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.
